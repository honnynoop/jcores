# SECI 모델 — 지식 변환의 네 가지 방식

> 노나카 이쿠지로(Nonaka Ikujiro)의 지식 창조 이론(Knowledge Creation Theory)에서 제시한 암묵지와 형식지 간의 네 가지 변환 방식

---

## 핵심 개념

| 구분 | 설명 |
|------|------|
| **암묵지 (Tacit Knowledge)** | 경험·직관·감각처럼 언어로 표현하기 어려운 개인의 내면적 지식 |
| **형식지 (Explicit Knowledge)** | 문서·데이터·매뉴얼처럼 언어나 기호로 표현된 공식적 지식 |

---

## SECI 모델 구조

```
              암묵지 (Tacit)          형식지 (Explicit)
            ┌─────────────────────┬─────────────────────┐
암묵지      │   S  공통화          │   E  표출화          │
(Tacit)     │   Socialization      │   Externalization    │
            │   암묵지 → 암묵지    │   암묵지 → 형식지    │
            ├─────────────────────┼─────────────────────┤
형식지      │   I  내면화          │   C  연결화          │
(Explicit)  │   Internalization    │   Combination        │
            │   형식지 → 암묵지    │   형식지 → 형식지    │
            └─────────────────────┴─────────────────────┘

순환 방향: S → E → C → I → S (나선형 반복)
```

---

## 네 가지 지식 변환 방식

### S — 공통화 (Socialization)
**암묵지 → 암묵지**

경험을 함께 나누며 말 없이도 지식을 전달하는 방식. 언어가 개입하지 않아도 지식이 이동한다는 점이 핵심이다.

- **방법**: 관찰, 모방, 체험, 현장 실습, 도제 관계
- **핵심 활동**: 같은 공간에서 함께 일하며 감각·노하우를 공유
- **예시**
  - 요리사 옆에서 직접 보고 따라하며 조리 감각 습득
  - 숙련 기술자와 함께 일하며 손기술을 체득
  - 현장 견학·인턴십

---

### E — 표출화 (Externalization)
**암묵지 → 형식지**

머릿속에 있는 노하우와 직관을 언어, 개념, 도형, 비유로 꺼내어 표현하는 과정. **SECI 모델에서 가장 중요하고 어려운 단계**로, 새로운 지식 창조의 핵심이다.

- **방법**: 개념화, 은유·비유 사용, 모델링, 가설 설정
- **핵심 활동**: 경험을 문서·매뉴얼·보고서로 명시화
- **예시**
  - 숙련 장인의 기술을 작업 매뉴얼로 정리
  - 전문가 인터뷰를 통해 암묵적 노하우를 지식 DB로 구축
  - 베스트 프랙티스(Best Practice) 문서화

---

### C — 연결화 (Combination)
**형식지 → 형식지**

이미 문서화된 형식지들을 수집·분류·편집·재구성하여 새로운 형식지를 만드는 과정. 기존 지식의 조합으로 새로운 가치를 창출한다.

- **방법**: 데이터 분류, 편집, 통합, 재구성
- **핵심 활동**: 여러 문서·데이터를 합쳐 새로운 시스템·보고서 생성
- **예시**
  - 각 부서 보고서를 통합해 경영 전략 계획서 작성
  - 여러 연구 논문을 종합한 메타 분석
  - ERP·데이터베이스 시스템 구축

---

### I — 내면화 (Internalization)
**형식지 → 암묵지**

문서·교재 등으로 학습한 형식지를 반복 실천을 통해 몸에 익혀 자신의 암묵지로 체화하는 과정.

- **방법**: 학습, 반복 훈련, 시뮬레이션, 실무 적용
- **핵심 활동**: 매뉴얼과 교육 내용을 실천하며 직관으로 습득
- **예시**
  - 매뉴얼을 숙지하고 반복 연습해 자동적으로 행동
  - 운전 교본을 배운 후 몸으로 운전을 익힘
  - OJT(On-the-Job Training)를 통한 업무 체화

---

## 순환 구조 — 지식의 나선형 확장

```
개인의 암묵지
     │
     ▼ [공통화 S]  경험 공유
조직의 암묵지 공유
     │
     ▼ [표출화 E]  언어·문서화
형식지 생성
     │
     ▼ [연결화 C]  통합·재구성
새로운 형식지 체계
     │
     ▼ [내면화 I]  실천·체화
더 깊은 암묵지
     │
     └──────────── 다시 순환 (더 높은 수준으로)
```

이 순환은 **개인 → 그룹 → 조직 → 조직 간**으로 범위가 확장되며 반복된다. 각 사이클마다 지식의 수준과 범위가 나선형으로 성장하는 것이 핵심이다.

---

## 비교 요약

| 구분 | 공통화 (S) | 표출화 (E) | 연결화 (C) | 내면화 (I) |
|------|-----------|-----------|-----------|-----------|
| 변환 방향 | 암묵지 → 암묵지 | 암묵지 → 형식지 | 형식지 → 형식지 | 형식지 → 암묵지 |
| 핵심 활동 | 경험 공유 | 언어·개념화 | 통합·재구성 | 실천·체화 |
| 주요 도구 | 현장, 대화 | 문서, 다이어그램 | DB, 보고서 | 훈련, OJT |
| 지식 주체 | 개인 간 | 개인 → 조직 | 조직 내 | 조직 → 개인 |
